Parameter C

이번 과제에서는 SVM 파라메터 중 하나인 C 값에 대해 살펴보겠습니다. 정의에 따르면 C는 정해진 서포트 벡터에 따라 나타나는 에러에 대한 페널티를 의미합니다. 또한, 각 학습 데이터에 대해서 얼마나 잘못 판단하는 것을 피할지에 대한 정도를 의미합니다. 즉, C에 큰 값을 할당하면, SVM은 최대한 학습 데이터를 맞추기 위해 노력합니다. 그렇게 되면 데이터에 맞게 잘 학습이 이루어질 것입니다. 하지만 이런 경우 학습 데이터에만 잘 학습이 되는 문제, 즉 과적합(overfitting) 상태가 발생할 확률이 높습니다. 반대로 적은 C값을 할당한다면 학습 데이터를 엉성하게 학습할 것입니다.

이번 과제에서는 그러한 C의 변화에 따른 학습 데이터에서의 정확도와 테스트 데이터에서의 정확도를 동시에 살펴보도록 하겠습니다.

과제

이번 과제에서는 헌혈 데이터를 이용하도록 하겠습니다.

  1. load_data() 를 이용해 헌혈 데이터를 읽어옵니다. 디렉토리에 blood_donation.csv 파일이 주어집니다. 파일 안 하나의 줄마다 한 헌혈자의 데이터가 기록되어 있으며, 각 항목은 ,로 구분되어 있습니다. pd.read_csv 를 이용해 손쉽게 데이터를 읽어들일 수 있습니다. 마지막 항목은 가까운 미래인 2007년 3월에 헌혈을 하였는지에 대한 정보, 즉 클래스로 0, 1의 형태로 되어 있습니다. 나머지 항목은 위의 4개 속성에 대한 값들이 적혀져 있습니다. 아래에 파일의 일부를 첨부하였습니다. 해당 파일을 읽어 X에는 각 데이터의 속성이 들어있는 행렬, y에는 각 데이터의 클래스가 들어있는 행렬을 만듭니다. X 및 y 를 만들 때는 pop 함수를 이용하면 쉽습니다. 지난 주 와인 데이터와 유사하오니 참고해주세요.
  2. 각 속성 별로 값들의 범위가 매우 다릅니다. 예를 들어 첫 번째 항목의 경우 일의 단위이지만, 세 번째 항목의 경우 천의 단위입니다. 따라서 이를 해소하기 위해 standardization 을 수행하는 것은 일반적입니다. 앞선 주차에서는 MinMaxScaler라는 함수를 이용하였습니다. 이번 과제에서는 좀 더 쉬운 sklearn.preprocessing.scale 함수를 이용하도록 하겠습니다. 해당 함수는 standardization을 원하는 데이터를 집어넣으면 이를 수행한 결과가 반환됩니다.
  3. run_rbf_SVM 함수를 구현합니다. 해당 함수는 데이터 전체를 학습한 SVM 모델을 반환합니다. 이 때, 커널로 rbf를 사용합니다.
  4. 학습된 SVM 모델의 정확도를 측정합니다. 이 때, 사용할 데이터는 X_data와 y_data 파라메터로 전달됩니다. 각 모델에 score 함수를 테스트 데이터를 파라메터로 전달하면서 실행시키면, 모델의 정확도 값을 반환합니다. 이를 반환하는 함수를 구현하시면 됩니다.
  5. 학습 데이터를 통해 학습된 SVM 모델의 정확도를 확인합니다. 이 때, 학습 데이터와 테스트 데이터를 동시에 이용하도록 하겠습니다. 학습 데이터를 이용한 테스트 결과를 train_score에 넣어주시고, 테스트 데이터를 이용한 테스트 결과를 test_score에 넣어주세요. 해당 문제에서는 test_svm_models 함수에 적절한 파라메터를 물음표를 없애고 넣으시면 됩니다.
